# SEO next steps and how to use the new tooling

This file describes the changes made and how to use them.

Files added/changed
- `public/og-image.svg` — 1200×630 Open Graph image used for rich previews.
- `public/robots.txt` — allows crawling and links to sitemap.
- `public/sitemap.xml` — initial sitemap (also generated by the script below).
- `public/manifest.json` — PWA manifest referencing `/t.png` icons.
- `scripts/generate-sitemap.js` — Node script to generate `public/sitemap.xml` from `app/` pages.
- `app/layout.tsx` — updated head: favicon, OG/Twitter tags, manifest link, verification placeholders, structured data retained.
- `package.json` — new scripts for generating sitemap and running Lighthouse audits.

How to generate the sitemap (recommended on deploy or locally)

1. From the repo root run:

```bash
npm run generate-sitemap
```

This will scan the `app/` folder for `page.tsx` files and write `public/sitemap.xml` (overwrites existing file).

Lighthouse audits (local)

1. Install dev dependencies (this will install `lighthouse` and `chrome-launcher`):

```bash
npm install
```

2. Start the dev server in another terminal:

```bash
npm run dev
```

3. Run the Lighthouse JSON audit (headless):

```bash
npm run audit:lighthouse
```

or HTML report:

```bash
npm run audit:lighthouse:html
```

Notes and verification

- To verify site ownership in Google Search Console add the verification meta tag into `app/layout.tsx` (replace YOUR_CODE):

```html
<meta name="google-site-verification" content="YOUR_CODE" />
```

- For Bing Webmaster Tools use:

```html
<meta name="msvalidate.01" content="YOUR_CODE" />
```

Add those tags in `app/layout.tsx` where placeholders are present, rebuild and redeploy.

Submitting sitemap to Google Search Console

- After verifying your site in Search Console go to the "Sitemaps" section and submit `https://tigattech.com/sitemap.xml`.

Automating sitemap on deploy

- Add the `npm run generate-sitemap` step to your CI/CD deploy pipeline (e.g., in GitHub Actions) before the build step so the sitemap is always up-to-date.

OG image

- `public/og-image.svg` is a simple templated image. For best results, produce a high-quality PNG (1200×630) and update `app/layout.tsx` meta tags to point to it.
